VPS サービスの品質に、業界共通の統一基準はありません。  
事業者は SLA・性能・サポートなどを組み合わせて品質を示しているため、ここではそれらを**比較の物差し**として整理します。

最初からすべてを満たす必要はありません。[[guides/vps/specs|必要スペック]] を満たすプランがあり、OS に **Ubuntu 24.04 以降** を選べれば始められます。

## 選定の観点

| 項目 | 評価内容 | 目安 |
| :-- | :-- | :-- |
| SLA | 稼働率・障害時の補償 | 99.9〜99.99% |
| 可用性 | 止まらないためのデータセンター設計 | 電源・回線の冗長など |
| 性能 | CPU・ストレージ・ネットワーク | NVMe・新世代 CPU など |
| サポート | 対応時間・手段 | メール / チャット / 電話など |
| セキュリティ | 事業者の認証・運用 | ISO 27001 など |

### SLA（サービス品質保証）

まず確認しやすいのが **SLA**（稼働率の保証）です。数字の差は小さく見えても、年間の停止時間は大きく変わります。

| SLA | 年間停止時間の目安 |
| :-- | :-- |
| 99.9% | 約 8 時間 46 分 |
| 99.95% | 約 4 時間 23 分 |
| 99.99% | 約 53 分 |

個人の学習用途では 99.9% でも十分始められます。差の意味を知ったうえで、用途に合わせて選べば問題ありません。

### 性能

品質は「速さ」も重要です。CPU・メモリ・SSD・ネットワーク帯域で評価します。  
**vCPU 数だけでは実効性能は決まりません。** CPU 世代や共有率、ストレージが NVMe SSD かどうかも確認しましょう。

コア数・メモリ量などの最低線は [[guides/vps/specs|必要スペック]] を参照してください。

### 可用性

品質の高い VPS は、電源の二重化、冗長ネットワーク、UPS などを備えたデータセンターで運用されています。  
手順というより、事業者選びの参考情報として捉えてください。

### サポート

障害時や設定で詰まったときに、誰に・どの手段で聞けるかは運用のしやすさに直結します。  
このガイドでは、**日本語のドキュメントや問い合わせ窓口があること**を重視します。

### セキュリティ・運用品質

ISO 27001 などの認証は、VPS 自体の速さではなく、**事業者の運用体制**を評価する材料です。あれば信頼性の参考になります。

## セルフホスティング向けの目安

必須条件ではなく、比較するときの目安です。Coolify を運用する場合は、とくにバックアップとディスク I/O（NVMe など）が効きやすくなります。

| 項目 | 目安 |
| :-- | :-- |
| SLA | あれば参考（個人用途は 99.9% でも可） |
| ストレージ | NVMe SSD が望ましい |
| バックアップ | スナップショットまたは外部バックアップ |
| サポート | 日本語の情報・問い合わせができること |
| 認証 | ISO 27001 などは信頼性の参考 |
| 価格 | 性能あたりのコストで比較 |

IPv6・逆引き DNS・API・24 時間サポートなどがあれば、さらに運用しやすくなります。

## 比較（実例）

上記の物差しを踏まえ、実際に利用したサービスの例を紹介します。

| 項目 | さくらVPS | シンVPS |
| :-- | :-- | :-- |
| SSH ログイン | `ubuntu` ユーザー → `sudo su -` で root 切替 | `root` ユーザーで直接ログイン |
| セキュリティ | 一般ユーザー経由のため望ましい | root 直接ログインはリスクが高い |
| Ubuntu バージョン | 24.04 提供（やや古い印象） | 最新版を選択可能な場合あり |

### さくらVPS

初回ログインは **`ubuntu` ユーザー** で行い、必要に応じて `sudo su -` で root に切り替えます。  
root で直接 SSH ログインできないため、ブルートフォース攻撃の対象になりにくく、セキュリティ上の利点があります。

Ubuntu 24.04 は提供されていますが、最新 LTS（26.04 など）が選べない場合があります。

- 公式サイト: [さくらVPS](https://vps.sakura.ad.jp/)

### シンVPS

**`root` ユーザーで直接 SSH ログイン** する仕様です。  
設定を簡略化できる反面、root アカウントが攻撃の標的になりやすいため、セキュリティ面では注意が必要です。

root 直接ログインを避けたい場合は、さくらVPS のように一般ユーザー経由で root に切り替える方式のプロバイダを選ぶことをおすすめします。

- 公式サイト: [シンVPS](https://www.shin-vps.jp/)

## 選び方のポイント

- **[[guides/vps/specs|必要スペック]]** を満たし、OS は **Ubuntu 24.04 以降** を選ぶこと
- SLA・バックアップ・サポートなどは比較の物差しとして使い、最初からすべてを満たす必要はないこと
- root 直接ログインより、一般ユーザー + `sudo` の方式を優先すること
- 契約・初期設定の詳細は各プロバイダの公式ドキュメントを参照すること
